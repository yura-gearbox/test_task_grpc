# Тестовое задание

Имеется программа на языке C++ (см. директорию cpp-application), выполняющая анализ сетевого траффика. Эта программа слушает UDP-порт 127.0.0.1:2032 и считает количество принятых датаграмм, а также количество байтов, соответствующих латинской букве 'A' в кодировке ASCII во всех принятых датаграммах с момента запуска программы. Затем она может как-то анализировать эти данные.

Требуется:

1. Составить тест-план для этой программы, проверяющий, что число датаграмм и букв 'A' в них считается правильно.
2. Автоматизировать выполнение данного тестирования.

## Инфраструктура

Программу требуется тестировать в окружении Ubuntu 24.04 LTS или старше.

Необходимо подготовить два скрипта bash:

1. build.sh, который собирает программу на C++ в директории build рядом с данным файлом.
2. test.sh, который запускает все необходимые тесты над собранной таким образом программой.

Необходимо задокументировать все шаги настройки окружения (установку датаграмм и т. п.).

## Требования к автотестам

Автотесты, запускаемые из test.sh, должны быть написаны на Python 3-ей версии, в виде тестов для фреймворка pytest. Отдельные проверки желательно, насколько возможно, разнести по разным тестам pytest, чтобы вывод о конкретных проваленных проверках был как можно более гранулированным.

Для взаимодействия с тестовой инфраструктурой тестируемая программа на C++ содержит GRPC-сервер на порту 127.0.0.1:2031, реализующий интерфейс monitor.MonitorService из файла cpp-application.monitor.proto. Получать актуальные значения счётчиков нужно вызовом `GetUdpStatistics()`.

Кроме того, интерфейс содержит метод `IsReady()`. Он возвращает флаг, указывающий на статус запуска программы. Пока этот метод не вернёт значение true, программа не готова анализировать траффик. После первого возврата true программа должна продолжать работать сколь угодно долго.

Протестировать, что программа запускается, переходит в состояние готовности, корректно считает датаграммы и символы 'A' в них, а также корректно завершается с нулевым кодом возврата.

По поводу дополнительной информации (тайм-ауты и т. п.) сделать правдоподобные предположения. Такие параметры могут меняться по мере разработки приложения и потому должны быть представлены в тесте так, чтобы их можно было легко изменить в будущем.

Для справки можно использовать исходный код программы на C++, однако тесты должны сохранять работоспособность при его изменении с сохранением поведения.
