# Автоматизация тестирования gRPC/UDP сетевого монитора

Проект представляет собой комплексное решение для системного тестирования C++ приложения `grpc_udp_monitor`. Написан на Python с использованием фреймворка `pytest` в изолированном Docker-контейнере.

## 🛠 Технологический стек

* **Язык автоматизации:** Python 3.12.3
* **Тестовый фреймворк:** pytest 9.0.3
* **Сетевое взаимодействие:** grpcio / grpcio-tools (1.81.1), socket (встроенный модуль)
* **Окружение:** Docker, Docker Compose, Ubuntu 24.04 LTS

## 🚀 Быстрый старт (Запуск в Docker)

Цикл тестирования полностью инкапсулирован внутри контейнера. Вся инициализация среды (создание `.venv`, установка зависимостей, компиляция `.proto`-файлов и генерация Python gRPC-стабов) происходит автоматически при старте контейнера.

### 1. Развертывание тестового контура

Выполните сборку образа и запуск контейнера на хост-системе:

```bash
docker compose up --build
```

Дождитесь завершения работы `pip install` внутри контейнера. После этого нажмите d (detach) либо перейдите в новое окно терминала и запустите контейнер в detach-режиме:

```bash
docker compose up -d
```

Для дальнейших действий откройте сессию терминала внутри контейнера:

```bash
docker compose exec test-env /bin/bash
```

Терминал автоматически откроется в рабочей директории `/workspace` с активированным виртуальным окружением, на что указывает префикс `(.venv)` в строке приглашения.

### 2. Сборка целевого C++ приложения

Запустите скрипт компиляции бинарного файла:

```bash
./build.sh
```

Скомпилированный исполняемый файл будет сохранен по пути `build/grpc_udp_monitor`.

### 3. Запуск автоматизированных тестов

Для прогона всего тестового сьюта (16 сценариев) выполните команду:

```bash
./test.sh
```

## 🎯 Избирательный запуск тестов по маркерам

Для удобства локальной отладки все автотесты размечены на логические группы с помощью встроенных маркеров pytest:

* **Проверка жизненного цикла (Smoke/Lifecycle):**
  
  ```bash
  pytest . -m smoke
  ```

* **Валидация спецификации счетчика пакетов:**
  
  ```bash
  pytest . -m packet_counter
  ```

* **Валидация спецификации счетчика целевых символов:**
  
  ```bash
  pytest . -m byte_counter
  ```

* **Тестирование аппаратных переполнений (White-Box):**
  
  ```bash
  pytest . -m overflow
  ```

* **Стресс-тестирование UDP-слушателя:**
  
  ```bash
  pytest . -m stress
  ```
